# Policy: Ramp (accessibility)

**Goal:** WCAG 2.1 AA remediation of the target site with zero visual regression.

The Builder may not mark any item green. Only Verifier A (scanner) and
Verifier B (vision) outputs flip an item from `[ ]` to `[x]`.

## Done means (all machine- or vision-verifiable)

- [ ] **axe-core: 0 violations** (serious + critical) on every crawled page, desktop and mobile — _Verifier A_
- [ ] **Keyboard-only**: every interactive element reachable in logical order; no focus traps; visible focus indicator on all — _Verifier A (focus-order/tabindex rules) + Verifier B (visible ring)_
- [ ] **Meaningful alt text** on every `<img>`, generated by LOOKING at the image, not the filename. Decorative images get `alt=""` — _Verifier B (vision)_
- [ ] **Color contrast** ≥ 4.5:1 body / 3:1 large text, verified by scanner, with brand hue preserved (adjust lightness, not hue) — _Verifier A + Verifier B_
- [ ] **Heading hierarchy** valid: one `h1`, no skipped levels — _Verifier A_
- [ ] **Forms**: every input has a programmatic label; errors announced via `aria-live` — _Verifier A_
- [ ] **Visual diff** within threshold on every page — contrast/focus-ring changes allowed, layout shifts rejected — _Verifier B_
- [ ] **Lighthouse accessibility** score ≥ 95 on every page — _Verifier A (optional, if Lighthouse available)_

## Scanner

axe-core via Playwright (`policies/a11y/scanner.mjs`), tags `wcag2a`, `wcag2aa`,
`wcag21a`, `wcag21aa`.

## Allowed visual deltas

color lightness · focus outlines · underlines on links

## Forbidden

layout shift · font changes · image swaps · content rewrites
