# Retrofit — demo run sheet (3 min)

One engine, two policies, run live. Every claim below maps to a real artifact in
this repo — nothing is hand-waved.

## Setup (once)

```bash
npm install
npx playwright install chromium chromium-headless-shell
npm run serve                                  # demo target on :8080
npx http-server reports -p 8088 -c-1 &         # dashboard on :8088
```

Open **http://localhost:8088/dashboard.html** — this is the whole story on one page.

## Beat 1 — the thesis (0:00)

> "Retrofit is a remediation engine that sees its own work. One engine, swappable
> policy files — we ran it twice today on two different problems."

Show the two cards at the top: **Ramp 32 → 0**, **Polyglot 63 → 0**, each with its
rubric green. Same engine, different `rubric.md`.

## Beat 2 — Ramp, the screen-reader flip (0:20)

Scroll to **"Screen reader: gibberish → coherent."** Before, the home page
announced `""`, `""`, `""`, `link ""`, `textbox ""`. After: every image and
control has a meaningful name — generated by **looking at the image**, not the
filename. Then point at the axe counter: **N → 0**.

Reproduce live:
```bash
npm run retrofit -- --policy a11y     # verify A + B + keyboard + axtree + rubric + report
```

## Beat 3 — the autonomy moment (1:20)

Show the amber callout under **Ramp — before / after**:

> Verifier B rejected the first `about.html` heading fix — re-tagging the title
> shifted the page ~12px (the new class set `line-height:1.2`; the original
> inherited `1.6`). **Nobody told it this broke — it saw it.** The Builder matched
> the line-height; it passed.

It's recorded in `NOTES.md` (Rejections log) — the engine's persistent memory, so
the mistake never repeats.

## Beat 4 — Polyglot, same command, different policy (1:50)

```bash
npm run retrofit -- --policy i18n
```

Show the site in Korean / German (the gallery, German card highlighted). Then the
second callout: the per-locale overflow check caught a bare `<img>` overflowing
mobile — **a defect the accessibility scanner never saw**, because axe doesn't
test horizontal overflow. Different policy, different findings.

## Beat 5 — close (2:30)

> "Done is a file the model grades itself against."

Show `policies/a11y/rubric.md` and `policies/i18n/rubric.md` side by side, then the
self-graded `reports/<policy>/rubric-status.md`. The Builder never flips an item —
only Verifier A (scanner) and Verifier B (vision) output does. **Swap the file,
get a new product. Anyone can rerun this tomorrow.**

## What grades what

| Rubric item | Verifier | Signal |
| --- | --- | --- |
| axe 0 violations | A | `verify-policy.mjs` re-scan |
| keyboard / focus | A | `keyboard.mjs` |
| meaningful alt | A + B | `axtree.mjs` + vision |
| contrast / headings / forms | A | axe rules at 0 |
| layout diff | B | `visual-capture.mjs` + `accepted.json` |
| 0 hardcoded strings | A | i18n lint + pseudo-locale |
| locale completeness | A | catalog scan |
| per-locale overflow | B | `locale-shots.mjs` |
| lang switcher | A + B | applied `lang` per locale |

## Regenerating the screen-reader "before"

The before/after AX dump serves the pre-remediation target from git history:
```bash
git archive 429a203 target | tar -x -C /tmp/before
npx http-server /tmp/before/target -p 8090 &
node harness/axtree.mjs --target http://localhost:8090 --label before --out reports/a11y/axtree-before.json
node harness/axtree.mjs --target http://localhost:8080 --label after  --out reports/a11y/axtree-after.json
```
